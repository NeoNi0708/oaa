- **Severity**: Critical — core feature broken
- **Category**: Functional
- **Fix Status**: Needs investigation. Likely LLM API timeout or agent loop silent failure.

### ISSUE-002 [HIGH] — Model selector not visible

- **Page**: Chat (对话)
- **Repro**: Navigate to chat page. No model dropdown visible in header area.
- **Evidence**: `snapshot -i` shows no `.model-selector` element in ARIA tree. Root cause identified in code review: `v-if="modelList.length > 0"` on a `Record` object always evaluates to `false`.
- **Severity**: High — users cannot switch between configured models
- **Category**: Functional
- **Fix Status**: Code fix applied in `ChatView.vue:9` (`Object.keys(modelList).length`) but needs rebuild restart to verify

### ISSUE-003 [MEDIUM] — WeChat QR login button flashes loading then resets

- **Page**: Connections (连接)
- **Repro**: Click "扫码登录" under WeChat card. Button briefly shows "生成中...", then returns to "扫码登录" with no visible error. No QR code displayed.
- **Evidence**: Backend returns error (iLink API unreachable) but frontend error is hidden by `v-if="qrCodeUrl"` guard. Error state visible only when QR section already rendered.
- **Severity**: Medium — feature non-functional but non-blocking (network-dependent)
- **Category**: Functional
- **Fix Status**: Backend error handling already present. Frontend error display relies on backend returning valid QR URL first — error shown only inside QR section which requires `qrCodeUrl`.

### ISSUE-004 [LOW] — Welcome hint chips not keyboard accessible
